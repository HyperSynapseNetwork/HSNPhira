CK服务器状态监控API服务文档

概述

本程序是一个独立的服务器状态监控服务，提供API接口返回服务器在线状态和延迟数据。程序每10分钟自动检查服务器状态，并记录最近48小时内的历史延迟数据。

系统要求

- Python 3.8 或更高版本
- 网络访问权限
- 服务器认证凭据

快速开始

1. 安装依赖

pip install flask aiohttp

2. 配置

USERNAME = "your_username"  # 服务器用户名
PASSWORD = "your_password"  # 服务器密码
API_PORT = 5000             # API服务端口
CHECK_INTERVAL = 600        # 检查间隔（10分钟，600秒）
HISTORY_HOURS = 48          # 历史数据保留小时数

API接口

获取当前服务器状态

- URL: 
"/api/status"
- 方法: GET
- 响应格式: JSON

响应示例:

{
  "online": /*布尔值，服务在线状态。true表示服务正常，false表示服务异常*/,
  "latency_ms": /*浮点数，接口响应延迟，单位为毫秒*/,
  "last_check": /*字符串，最后一次健康检查时间，ISO 8601格式*/,
  "server_name": /*字符串，服务器名称标识*/,
  "timestamp": /*字符串，当前接口响应时间，ISO 8601格式*/
}

获取历史延迟数据

- URL: 
"/api/history"
- 方法: GET
- 参数:
   - 
"hours" (可选): 指定返回多少小时的数据，默认为48小时
- 响应格式: JSON

响应示例:

{
  "server_name": /*字符串，服务器名称标识，与健康检查接口一致*/,
  "data_points": [
    {
      "timestamp": /*字符串，数据点时间戳，ISO 8601格式*/,
      "online": /*布尔值，该时间点的服务在线状态。true表示在线，false表示离线或异常*/,
      "latency_ms": /*浮点数或null，该时间点的接口响应延迟，单位为毫秒。当online为false时可能为null*/,
      "error_message": /*字符串或null，当online为false时的错误描述信息。可选字段，仅在线状态异常时出现*/
    }
  ],
  "summary": {
    "total_points": /*整数，总数据点数*/,
    "online_points": /*整数，在线状态的数据点数*/,
    "offline_points": /*整数，离线状态的数据点数*/,
    "availability_rate": /*浮点数，服务可用率，单位为百分比（0-100）*/,
    "avg_latency": /*浮点数，平均延迟，单位为毫秒。仅计算在线状态的数据点*/,
    "max_latency": /*浮点数，最大延迟，单位为毫秒。仅计算在线状态的数据点*/,
    "min_latency": /*浮点数，最小延迟，单位为毫秒。仅计算在线状态的数据点*/
  }
}

服务健康检查

- URL: 
"/api/health"
- 方法: GET
- 响应格式: JSON

技术架构

程序采用分层架构设计：

1. API层: Flask框架提供RESTful API接口
2. 业务层: ServerStatusMonitor类管理监控逻辑和历史数据
3. 数据层: 内存存储当前状态和历史数据
4. 网络层: aiohttp处理异步HTTP请求

核心功能

定时监控

- 每10分钟自动执行服务器状态检查
- 异步非阻塞设计，不影响API响应
- 异常重试和错误处理机制

历史数据记录

- 自动记录每次检查的结果
- 保留最近48小时的数据（可配置）
- 自动清理过期数据
- 提供数据统计和摘要信息

状态检查流程

1. 登录认证服务器获取访问令牌
2. 连接目标服务器验证连通性
3. 测量网络延迟并记录结果
4. 存储结果到历史记录
5. 清理过期数据

配置参数

参数 说明 默认值
USERNAME 服务器认证用户名 必填
PASSWORD 服务器认证密码 必填
CHECK_INTERVAL 状态检查间隔（秒） 600（10分钟）
HISTORY_HOURS 历史数据保留小时数 48
API_PORT API服务监听端口 5000
API_HOST API服务监听地址 0.0.0.0

历史数据格式

历史数据以列表形式存储，每个数据点包含以下字段：

{
  "timestamp": "ISO格式时间戳",
  "online": "布尔值，表示服务器是否在线",
  "latency_ms": "浮点数，延迟毫秒数（服务器离线时为null）",
  "error_message": "字符串，错误信息（仅当有错误时存在）"
}

数据统计

历史接口提供以下统计信息：

1. 总数据点数: 返回时间段内的总检查次数
2. 在线点数: 服务器在线的检查次数
3. 离线点数: 服务器离线的检查次数
4. 可用率: 服务器在线比例（百分比）
5. 平均延迟: 在线状态下的平均延迟
6. 最大延迟: 在线状态下的最大延迟
7. 最小延迟: 在线状态下的最小延迟
